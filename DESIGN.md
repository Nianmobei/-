## 一、项目概述

**兵戮灾 · 红之纷争**是一款 9×9 棋盘双人兵棋推演游戏，热座对战（同机轮流操作）。  
两名玩家分别控制**红骑士团**（铁卫系，有序战争）和**灾兽群**（散兽系，混乱传播），轮流规划行动，同步结算。  
胜利条件：**歼灭对方全部单位** 或 **连续占领对方本阵 2 回合**。

### 世界观背景

> 「我是红，纷争之红，战争之红。」

**红**是第一纪的历史学者，钻研战争之术后收纳了所有与暴力相关的星，将其破碎制造出"灾兽"以补强自身概念。坠日之战后，祂的两个化身——**红骑士**（有序战争）与**灾兽**（混乱传播）——爆发终战并同归于尽，留下七百年战乱余波。此即第二纪终结大灾之一：**兵戮灾**。

- **红骑士团**：有序的战争力量，能以任何形式制造兵团，擅长列阵防卫
- **灾兽群**：混乱的传播力量，侵蚀与噬灭，擅长机动袭扰

> ⚠️ 红骑士团与灾兽群的成员彼此厌恶，这是设计上两派不可共存的世界观来源

1. **决策密度高**：每回合移动 + 行动两个决策，行动有攻/防/占领三选项
2. **信息透明**：规划阶段可预览攻击箭头，知道会打谁
3. **前后差别明显**：动画阶段有严格串行的攻击演出，不同时播放
4. **进化作为节奏控制**：慢速进化（3回合奖励机制），进化选择作为关键决策点
5. **地形改变战术**：3格永久地形，战壕防守、高地进攻各有适用场景

### 特性说明

| 特性 | 触发时机 | 效果 |
|------|---------|------|
| **列阵** | 未移动时被攻击 | 减伤 1 |
| **列阵协同** | 己方列阵单位击杀敌人时 | 相邻己方列阵单位 +1 经验 |
| **噬溃** | 本回合攻击目标死亡 | 自身回血 1（仅自己击杀才触发） |
| **硬壳** | 任何时候被攻击 | 常驻减伤 1 |
| **铁壁** | 不动的铁壁光环单位相邻时 | 相邻友方减伤 1（光环，需自身不动） |
| **劲弩/投射** | 配置 `ranged=True` | 射程 2，但移动后本回合不能远攻 |
| **战旗** | 距旗卫曼哈顿距离 ≤ 2 | 同阵营单位攻击 +1 |
| **齐射** | 旁边有友军 | 远程单位攻击 +1 |
| **冲阵** | 直线移动时遇敌 | 直线冲锋最多 spd 格，遇敌停止并造成伤害 |
| **威压** | 攻击后 | 目标下回合无法移动 |
| **集群** | 攻击命中时 | 对目标相邻敌方各造成 1 伤 |

`can_make=False` 全表设置。`_build_act_buttons` 里 `can_make=True` 才显示"制造"按钮，当前不可用。

## 五、回合结算流程

### 操作顺序（main.py → ui.py）

```
红方规划 (p1_plan) → 确认 → 遮挡提示 (p1_done) → 灾方接手 → 灾方规划 (p2_plan)
→ 确认 → 执行 (execute_turn) → 动画 (animating) → 结算 (result)
→ 进化弹窗 → 新回合 (new_round)
```

### execute_turn 内部顺序（game_engine.py）

```
1. 重置 last_attack_events = []
2. _phase_move：移动阶段
   2a. 战车冲阵预处理（冲阵特性单独处理）
   2b. 计算意图位置 intent[uid] = (x,y)
   2c. 碰撞处理（敌方同格：速度高者留，低者退）
   2d. _resolve_collisions：碰撞互攻结算
3. _phase_action：行动阶段
   3a. 标记防御单位 u.defending=True
   3b. 攻击者列表，random.shuffle 后按速度降序排
   3c. 主攻击循环：每个攻击者打最近未处理目标，记录 attacks_received
   3d. 结算主攻击伤害（pending_dmg 刷新）
   3e. 反击阶段：防御中→反击所有攻击者；未防御→反击第一个攻击者
   3f. 噬溃触发（仅检查自己打死的目标）
   3g. 列阵协同触发
   3h. 威压触发
   3i. 集群溅射触发
4. _flush_dead：hp≤0 标记死亡
5. 驻扎回血（营地≥2级才生效）
6. 本阵占领检查
7. _check_evolution：进化判定
8. _check_victory：胜利判定
9. _update_camp_level
10. turn += 1
```

### 伤害计算公式（_calc_damage）

```python
atk = unit.base_atk
atk += 1  # 若在高地
atk += 1  # 若有战旗友军距离≤2（最多加1次）
atk += 1  # 若有齐射且旁边有友军

def_bonus = 0
def_bonus += 1  # 硬壳
def_bonus += 1  # 列阵且未移动
def_bonus += 1  # 铁壁友军在旁且未移动
def_bonus += 1  # 战壕地形

dmg = max(0, atk - def_bonus)
```

> ⚠️ **防御行动（ACT_DEFEND）不再提供减伤**，改为反击机制（见下）

### 反击机制

| 状态 | 反击范围 |
|------|---------|
| 本回合选择"防御" | 反击**所有**攻击过自己的单位 |
| 本回合未选择防御 | 只反击**第一个**攻击过自己的单位 |

反击伤害也走 `_calc_damage(tgt, atk, cur_alive)` 完整计算，包含地形和特性加成。  
反击事件追加到 `last_attack_events`，但**噬溃只检查主攻击阶段事件**（用 `main_event_end` 边界索引隔离）。

> ⚠️ **重要 Bug 记录**：反击事件追加到 `last_attack_events` 后，噬溃如果遍历全部事件，  
> 反击击杀也会触发回血，相当于散兽每回合双倍噬溃机会，导致主配对从 50% 跌到 34%。  
> 修复方式：在反击阶段前记录 `main_event_end = len(self.last_attack_events)`，  
> 噬溃检查改为 `self.last_attack_events[:main_event_end]`。

## 六、进化系统

### 经验获取途径

1. **战斗击杀**：直接击杀敌方单位 → `u.kills += 1`
2. **本阵占领**：占领本阵时1级单位 → `u.kills += 1`（BaseState.update）
3. **列阵协同**：列阵单位击杀后，相邻列阵友军 → `nb.kills += 1`
4. **每3回合奖励**（`_check_evolution`，`turn % 3 == 0`）：
   - 每方经验最高且未达门槛的非胚体单位额外 +1 经验

### 进化门槛

| 升级 | 所需经验（kills） |
|------|----------------|
| 1级 → 2级 | 1 |
| 2级 → 3级 | 3 |

### 进化弹窗流程（ui.py → _open_evo_popup）

1. 回合结算后 `ui.queue_evolutions(game)` 收集有 `_pending_evo` / `_pending_evo3` 属性的单位
2. 逐个弹出悬浮面板（贴近单位，自动判断左/右侧）
3. 3个按钮：路线A、路线B、不进化
4. 无论选哪个，**HP 全部补满**（`evolve_unit` 中 `u.hp = u.max_hp`，`skip_evolution` 中同样）
5. `EVO_TABLE` 定义各兵种的两条路线名和描述（ui.py 顶部）

### 注意事项

- `_pending_evo` = 1→2 待进化标记，`_pending_evo3` = 2→3 待进化标记
- 这两个是动态属性（`setattr`），用 `hasattr` 检查，`clear_turn_state` 时清除
- 如果 EVO_TABLE 没有对应条目（无进化路线），直接调用 `skip_evolution` 跳过

## 七、动画系统

### 阶段流水线（animator.py → AnimPhase）

```
IDLE → MOVE(0.30s) → SETTLE(0.10s)
     → [LUNGE(0.20s) → RECOIL(0.18s) → GAP(0.06s)] × N对（严格串行）
     → RANGED(0.38s)
     → DAMAGE(0.70s) → DONE
```

### 关键设计

- **非战斗单位**（不在 `last_attack_events` 中的）在 MOVE 阶段正常滑动
- **战斗单位**在 MOVE 阶段保持原位，SETTLE 后才落到最终位置
- **每对战斗**独立 LUNGE→RECOIL 演出，`_pair_idx` 逐对推进，严禁同时播放
- **LUNGE**：攻击方向目标冲刺 0.28 格（`sin(t*π)` 曲线），回弹
- **RECOIL**：防守方向弹退 0.18 格（`sin(t*π)` 曲线），闪白
- **远程单位**：在 RANGED 阶段播放抛射体（弩箭=直线蓝白+尾迹；炮弹=抛物线橙黄+光晕）
- **震屏**：有死亡时触发，强度 = `2.0 + 死亡数×1.8`，持续 0.26s

### 伤害数字

- `DamageTag`：记录 uid、伤害量、格子坐标
- DAMAGE 阶段渲染，向上漂浮 42px，逐渐淡出
- 治疗（HP上升）显示绿色 `+N`

## 九、UI 操作逻辑

### 规划阶段操作

1. **点击/拖拽**己方单位 → 选中，显示移动范围（蓝色格）和攻击范围（红色格）

- 悬浮效果：鼠标悬停按钮变色（`evo_hover`）
- 点击任意一个选项后立即推进到下一个待进化单位

## 十一、已知边界与注意事项

### 战斗边界

| 情况 | 当前处理 |
|------|---------|
| 攻击者和目标同时死亡 | `_check_victory` 用HP总量判胜负 |
| 反击时攻击者已死 | `if not atk or atk.dead: continue` 跳过 |
| 远程单位本回合移动 | `if atk.ranged and atk.did_move: continue` 不能远攻 |
| 碰撞后已处理的对 | `processed` frozenset 跳过，远程仍可另找目标 |
| 冲阵遇多个敌人同格 | 对所有人造成冲阵伤害后停止 |

### 动画边界

| 情况 | 当前处理 |
|------|---------|
| 反击事件进入 `last_attack_events` | 会生成额外 CombatPair，可能有多余动画对 |
| 同一对 (atk,tgt) 出现多次 | `seen` set 去重，只取第一次 |
| 死亡单位的动画 | DAMAGE 阶段淡出（`dead_alpha`），200→0 |

> ⚠️ **潜在问题**：反击事件也追加到 `last_attack_events`，导致 animator 的 `_pairs` 会包含反击对。如果反击造成的动画过多，可在 `setup()` 里过滤 `is_collision==False` 且 event 索引 > 主攻击数量之后的事件。目前因为5v5人数少，表现可接受。

### 进化边界

| 情况 | 当前处理 |
|------|---------|
| 单位进化后下回合立即死亡 | 正常，已经补满血 |
| 多个单位同时到达进化门槛 | `queue_evolutions` 队列化，逐个弹窗 |
| EVO_TABLE 无条目（如胚体进化） | 直接调用 `skip_evolution` |
| 进化弹窗时 `evo_popup_rect` 未初始化 | `renderer._draw_evo_popup` 有 `if not hasattr` 守卫 |

### UI 边界

| 情况 | 当前处理 |
|------|---------|
| 动画播放中点击 | `phase="animating"` 时 `handle_event` 直接 return None |
| 进化弹窗期间点击棋盘 | `if self.evo_uid is not None` 优先走 `_handle_evo_event` |
| 拖拽到格子外松手 | `cell=None`，不设置 planned_dir，保持原规划 |

## 十二、平衡历史与设计决策

### 为何铁卫 HP=4 而非 3？

- 列阵（不动减伤1）和噬溃（击杀回血1）表面等效，但实际不平衡：
  - **列阵**需要放弃机动性（不移动），且只在被攻击时才有效
  - **噬溃**主动进攻就能触发，且有滚雪球效应（越强越回血）
- 实测：HP=3时列阵胜率38%以下，HP=4时50%
- 结论：被动防御型特性需要更高的基础HP来体现价值

### 为何防御改为反击而非减伤？

- 原"防御=+1防"使防御永远不如攻击，没有战术意义
- 新机制："防御=反击所有攻击者"给防御一个独特的战术场景：
  - **正确使用**：在被多单位夹攻时防御，用一个行动反击所有人
  - **不防御**：只反击第一个，但可以自由移动+主动攻击
- 这创造了真正的决策树，而不是"永远选攻击"

### 为何每3回合奖励而非每2回合？

- 每2回合：第2回合首次触发，所有单位进化前尚未接触，过快失去意义
- 每3回合：给战斗时间积累自然经验，奖励锦上添花而非主要来源

### 为何噬溃改为回血1（原为2）？

- 回血2时：每次击杀+2HP，5只散兽可形成强滚雪球，散兽胜率>65%
- 只检查"自己打死"而非"有敌方死亡"是为了防止搭便车触发

1. `unit.py` → `UNIT_TEMPLATES` 加兵种，`Unit.has_trait` 自动支持
2. `game_engine.py` → `_calc_damage` 加攻防修正，`_phase_action` 加特效触发
3. `ui.py` → `EVO_TABLE` 加进化路线选项
4. `animator.py` → 如需新动画类型，扩展 `AnimPhase` 和 `update`
5. `renderer.py` → 在 `_draw_single_unit` 或新方法中添加视觉表现

### 关键变量速查

| 变量 | 位置 | 含义 |
|------|------|------|
| `u.kills` | Unit | 经验值（击杀数），控制进化门槛 |
| `u.defending` | Unit | 本回合是否防御（回合结束清空） |
| `u.did_move` | Unit | 本回合是否已移动（影响列阵减伤） |

| `anim._pairs` | AnimationManager | 近战对队列，串行播放 |
| `ui.evo_uid` | UIState | 当前正在处理进化的单位 uid，非 None 时锁定操作 |
| `ui.pending_evo_uids` | UIState | 待进化队列 |
